// 02 · Algorithm Design

Cost-aware
greedy routing.

01
Priority Scoring & Sort
Each transaction is scored: priority×10 + urgency×8 + value×2. Tight deadlines and high-value transactions are scheduled first to prevent costly delays.
02
Earliest Slot Search
For each channel, find the earliest start_time ≥ arrival_time within deadline arrival + max_delay, respecting concurrent capacity via interval overlap [start, start+lat).
03
Minimum Cost Selection
All three channels are evaluated in parallel. The channel minimizing fee + 0.001×amount×delay wins. The slot is booked immediately.
04
Graceful Failure Check
If the best routing cost exceeds 0.5×amount, the transaction is explicitly marked failed — because sometimes failing is cheaper than routing with extreme delays.
// cost_model.py
# Per settled transaction delay = start_time − arrival_time delay_penalty = 0.001 × amount × delay tx_cost = channel_fee + delay_penalty # Per failed transaction fail_cost = 0.5 × amount # Judge metric (minimize this) total_cost = Σ(tx_cost) + Σ(fail_cost)
# Sort key (descending) score = priority × 10 + (1/max_delay) × 8 + (amount/20000) × 2
Complexity: O(n log n) sort + O(n·k·c) slot search
k = 3 channels  |  c = concurrent slots
Handles 1000-tx stress datasets in milliseconds.
